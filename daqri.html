SECTION 2: Change Details

BUILD NUMBER: 006

Embed SecureBoot keys in the Coreboot

Author: Ranga

Steps to embed SecureBoot keys:

  • Edit CorebootPayloadPkgIa32X64d.dsc:
    SECURE_BOOT_ENABLE = TRUE
  • Set coreboot-acela\configs\config.kblrvp3\CONFIG_SB=y.
  • Put the keys db.auth, KEK.auth, PK.auth in coreboot-acela.

Testing

  • Boot to Ubuntu and run flashrom to flash coreboot.rom into SEC BIOS (with keys inside).
  • Let EC remain at gpioset bios_sw 1.
  • Reboot and verify OS fails to boot as grub.efi is not signed.
  • Boot to shell and run the signed applications.

Author: Ranga

EFI Shell

  • Change the BootOrder from EFI.
  • EfiShell> dmpstore -d "Variable"
  • dmpstore mtc (higher 16 bits will have the number of resets since the board powered-on).

OS

  • efibootmgr -o 0 1 2 3 4 (change boot order).
  • efibootmgr -O bootorder (delete boot order).
  • efibootmgr -c -L "label" -l "media/acela/USB/EFI/BOOT/bootx64.efi"

Notes

Build steps:
  1. Build the coreboot-acela with 4.8.1.
  2. Merge 4.6 Acela changes to Coreboot 4.8.1.
  3. Configure the Acela project:
    make defconfig KBUILD_DEFCONFIG=configs/config.kblkrvp3
  4. Setup the tools:
    make crossgcc-i386 CPUS=6
  5. Build Coreboot:
    make CPUS=6
  6. Pick the binary:
    coreboot-4.8.1/build/coreboot.rom
  7. Flash it:
    flashrom -w coreboot.rom
